二、eco-pipeline：总指挥
================================================================

输入："研究主题或auto" + "项目目录路径"

阶段0（topic="auto"时）
  → 调用eco-idea发现研究问题
  → 读ECO_IDEA.md，取推荐星级最高的候选
  → 向用户确认后进入阶段1

阶段1：初始化 ECO_BRIEF.md
  · 记录主题/目录/数据路径
  · 流水线状态追踪（全程更新）

阶段2：调用 eco-data-explore
  → 读ECO_DATA_PROFILE.md，提取
  · ID变量、时间变量
  · 候选Y/X/控制变量
  · 识别策略初判、FE组合、聚类层级
  · 是否需要Wild Cluster Bootstrap

阶段2.5：切口具体性关卡（必过，否则不推进）
  4道检查，每道1分：
  □ X是具体命名事件（非水平变量）
  □ 有明确冲击年份
  □ 处理单元是微观个体（企业/县/乡镇）
  □ 有自然对照组
  ≥3分 → 继续
  <3分 → 强制缩窄话题，搜索具体政策事件，重评
  0分  → 停止流水线，要求重新定义切入点

  切口改造示例：
    宽：金融发展如何影响企业创新？
    窄：2014年某市设立的金融改革试验区如何影响区内企业专利申请？

阶段3：调用 eco-lit-review
  → 读ECO_LIT.md，提取
  · 推荐识别策略
  · 机制变量清单
  · 异质性分组清单
  关键检查：文献空白不足 → 建议缩窄角度

阶段4：调用 eco-analysis
  执行前确认所有变量已从状态文件获取
  异常处理：主系数不显著 → 如实报告结果，不自动换题/换变量
            （自动换题是 specification searching，损害学术规范）
            建议用多结果（Table 2b）、置信区间、异质性分析建立论证
            若需调整研究问题，须由用户主动确认后再切换

阶段5：调用 eco-write-paper
  确认tables/目录存在且含所有表格

三、eco-idea：找研究问题
================================================================

输入：数据路径 + 研究背景（可选）

步骤1：AER发表潜力评估标准（内嵌）
  ──────────────────────────────────────
  识别策略可行性（0-5分）
    5分：有明确政策冲击，可做DiD
    4分：有工具变量候选（地理/历史/Bartik）
    3分：面板TWFE+时间趋势控制
    1-2分：截面OLS/纯相关性

  切口具体性（0-5分）※顶刊核心门槛
    5分：具体命名事件 + 明确年份 + 微观处理单元
    4分：可识别政策类别 + 年份范围 + 城市/县级
    3分：政策背景模糊 or 省级处理单元
    2分：宏观趋势变量，无明确冲击
    1分：两个宏观总量变量的相关性

    切口太宽的警报信号（降至1-2分）：
    · X是"水平/程度/指数"而非"事件/冲击"
    · 处理单元是国家或省级（N<50）
    · 没有明确的政策实施年份
    · 已有5篇以上同类宏观研究

  新颖性（0-5分）
    +2：因果识别新颖（现有文献多为相关性）
    +1：情境新颖（中国/发展中国家）
    +1：机制新颖（未检验具体传导机制）
    +1：异质性新颖（忽略某类分组差异）

  综合评分（满分15）
    ≥12 ★★★★★ 强烈推荐
    10-11 ★★★★ 推荐但需确认切口
    8-9  ★★★  建议缩窄后推进
    <8   ★★以下 不推荐

步骤2：Python读取数据
  · 识别ID列、时间列
  · 统计缺失率
  · 找ln_开头候选变量

步骤3：循环TWFE扫描（核心）
  · 所有ln_变量两两配对
  · feols(y ~ x | id + time, CRV1)
  · 筛选|t|>2.0
  · 输出ECO_IDEA_SCAN.csv + ECO_IDEA_SIGNIFICANT.csv

步骤4：新颖性筛选
  排除：GDP→GDP / 规模→规模 / 金融发展→增长 / 贸易→GDP
  保留：非传统X / 有溢出特征 / 有政策冲击时序

步骤5：切口具体性 + 识别策略综合评估
  【A】切口评分（4项）
  【B】切口<3时，强制WebSearch找具体政策事件
       搜索：[变量含义] [年份范围] 政策改革 中国
  【C】识别策略评估（遗漏变量/反向因果/工具变量）

步骤6：找数据断点
  · 各变量年度均值跳变检测
  · 跳变>5% → WebSearch确认对应政策
  · 是最强DiD识别机会

步骤7：写 ECO_IDEA.md
  每个候选必须包含：
  · 一句话切口：[年份]+[具体政策]+[微观单元]+[Y]
    示例："2016年缅甸矿业暂停令 → 乡镇级武装冲突下降69%"
  · 切口具体性得分（≥4才推荐）
  · 政策冲击名称/年份/覆盖范围/处理组/对照组
  · 综合评分和推荐星级

四、eco-data-explore：读懂数据
================================================================

输入：数据路径

步骤1：Glob扫描 .xlsx/.csv/.dta

步骤2：Python读取
  · 行列数、变量名、数据类型、描述统计、缺失率、前3行

步骤2b：数据清洗预处理
  ──────────────────────────────────────
  2b-1 日期格式检测转换
    · 检测字符串日期（YYYY-MM-DD/YYYYMMDD/YYYY/MM/DD）
    · 匹配度>80%即确认
    · pd.to_datetime解析 → 生成_year、_month列
    · 报告解析失败行数

  2b-2 同主体同年份多观测检测与聚合
    · groupby(ID, year).size() 检测重复
    · 最大重复数 + 重复比例
    · 若max>1：生成df_agg（数值列sum聚合）
    · 展示示例对比（原始 vs 聚合）
    · 不自动覆盖，等用户确认

步骤3：识别关键变量类型
  · 面板结构：ID列 + 时间列
  · 候选Y：产出/创新/金融类结果变量
  · 候选X：政策虚拟变量/基础设施/制度变量
  · 候选Z：GDP/人口/资本等控制变量

步骤3b：识别策略初判
  · 检测1：treat/treated/post/policy关键词 → DiD信号
  · 检测2：年份均值跳变>2倍平均波动 → RD/断点信号
  · 检测3：geo/dist/rainfall/hist关键词 → IV信号
  · 输出：主推策略 + 备选策略 + 疑似断点年份

步骤3c：固定效应推荐
  · 个体+时间+行业 → 个体FE + 行业×年FE
  · 个体+时间 → TWFE
  · 仅截面 → 地区/行业FE
  · 输出pyfixest公式示例

步骤3d：聚类SE推荐
  · 唯一值<50 → 警报：需Wild Cluster Bootstrap
  · 输出推荐聚类变量

步骤4：写 ECO_DATA_PROFILE.md
  · 数据集概览、面板结构
  · Y/X/Z变量表（变量名/含义/类型/缺失率）
  · 完整描述统计
  · 数据质量注意事项（含清洗结果）
  · 识别策略初判、FE组合、聚类层级、断点年份

五、eco-lit-review：调研文献
================================================================

输入：研究主题

步骤1：拆分研究主题
  · X是什么，Y是什么
  · 内生性来源
  · 潜在外生变动
  · 研究背景

步骤2：6轮WebSearch
  轮1：顶刊直接相关（aeaweb/qje/restud）
  轮2：NBER工作论文
  轮3：DiD/IV/RD识别策略参考
  轮4：中国情境（如适用）
  轮5：机制相关
  轮6：综述类（JEL/JEP）
  每篇记录：作者/年份/期刊/识别策略/主系数/数据来源

步骤3：归纳3-5条文献脉络
  脉络1：直接相关（X→Y）
  脉络2：机制相关
  脉络3：类似识别策略
  脉络4：本土情境
  脉络5：异质性相关

步骤4：推荐识别策略
  策略A（首选）：DiD → 外生变动/处理组/对照组/平行趋势论证
  策略B（次选）：IV → 工具变量候选/相关性/排他性
  策略C（备选）：Panel FE + 时间趋势

步骤5：识别研究空白
  · 因果识别空白
  · 地理/情境空白
  · 机制空白
  · 异质性空白
  · 数据空白

步骤6：提出可检验假设
  H1：主效应假设
  H2：机制假设
  H3：异质性假设

步骤7：写 ECO_LIT.md
  · 文献脉络、研究空白、识别策略
  · 机制变量清单（→ eco-analysis步骤6）
  · 异质性分组清单（→ eco-analysis步骤7）

六、eco-analysis：跑实证
================================================================

输入：研究主题 + 数据路径
读取：ECO_DATA_PROFILE.md + ECO_LIT.md

步骤0：提取所有分析变量
  基础变量：
  · ENTITY_VAR（个体ID）
  · TIME_VAR（时间）
  · Y_VAR（主被解释变量）
  · X_VAR（核心解释变量）
  · CONTROL_VARS（控制变量列表）
  · MECHANISM_VARS（机制变量）
  · HETEROGENEITY_VARS（异质性分组变量）
  · STRATEGY / HAS_DID / HAS_IV

  新增变量：
  · OUTCOME_VARS（理论驱动多结果Y列表，3-6个）
  · SPILLOVER_VAR（毗邻/关联对照单元虚拟变量）
  · ALT_ACCOUNTS（替代解释变量列表）

步骤1：数据读取+清洗
  · 自动识别.dta/.xlsx/.csv
  · listwise deletion（去掉Y和X缺失行）
  · 对数变换（正值变量生成ln_前缀列）
  · 缩尾1%（生成_w后缀列，稳健性时启用）

步骤2：Table 1 描述统计
  · 全样本：N/均值/SD/P25/P75
  · DiD时必做基线平衡检验：
    - 处理组 vs 对照组均值差 + t检验p值
    - 联合F检验（所有前定变量应联合不显著）

步骤3：Table 2 基准回归（含结构化安慰剂列）
  列1-6：递进规格（无FE → 控制变量 → 个体FE → 双向FE → DiD → ln(Y)）
  安慰剂列A：更高地理层级处理变量（县级政策 → 用市级处理），系数应不显著
  安慰剂列B：时间安慰剂（将政策提前2期，全样本），系数应不显著
  安慰剂列C（可选）：地理不变量（海拔/降水）作为安慰剂X
  · 安慰剂直接进Table 2，让审稿人一眼看到识别干净
  · 聚类数<50 → 触发WCB警告
  · 提取主模型β/SE/t/效应%

  Table 2b 多结果变量综合表（顶刊核心格式）
  · 固定主规格（双向FE+控制变量）
  · 对OUTCOME_VARS每个Y各跑一列
  · 展示事件影响的广度（哪些Y显著，哪些不显著）
  · 不显著的Y可作为安慰剂型Y（有助排除替代解释）
  · 输出tables/table2b_multi_outcomes.txt + .csv

步骤4：Table 3 识别策略
  ──────────────────────────────────────
  分支A：DiD + 事件研究（有DiD时）
  · 先检测是否为 Staggered Adoption（各单元首次处理年份是否不同）
  · 若 Staggered：
    - 主报告：Callaway & Sant'Anna (2021) CS-DiD（csdid包）
      · Simple Aggregate ATT + Dynamic Aggregate（事件研究图）
      · 输出tables/table3_csdid.csv + figure1_event_study.pdf/png
    - 对比：TWFE事件研究（仅作参考，注明存在负权重问题）
  · 若单一处理时间：
    - 标准TWFE事件研究（k=-max_lead到+max_lag，k=-1基准期）
    - 平行趋势联合Wald检验（超前期系数联合应p>0.1）
    - 输出tables/table3_event_study.txt + figure1_event_study.pdf/png

  分支B：IV/2SLS（有工具变量时）
  · 第一阶段：F统计量
    - F>10：Stock-Yogo经验值
    - F>104.7：Montiel Olea-Pflueger最严格标准
  · 2SLS主回归
  · OLS vs IV系数对比（判断内生性方向）
  · 输出tables/table3_iv.txt

步骤5：Table 4 稳健性检验
  ──────────────────────────────────────
  通用5项（所有策略必做）：
  ① 缩尾Winsorize 1%
  ② 替换Y（对数化）
  ③ 放松FE（只加个体FE，去掉时间FE）
  ④ 无控制变量（加强FE）
  ⑤ 剔除极端X（5%-95%样本）

  IV专项：
  · Hansen J过度识别检验（多IV时）

  · 输出tables/table4_robustness.txt

  Table 4b 事件驱动宏观结果扫描
  · 逻辑：固定外生事件X，扫描数据中所有候选Y
  · 自动排除：X本身/ID/时间/控制变量/ln_前缀/缩尾列/缺失>50%
  · 对每个候选Y跑相同规格（双向FE+控制变量+CRV1）
  · BH-FDR多重检验校正（q<0.1）
  · 输出tables/table4b_outcome_scan.csv

  Table 4c 替代解释排除（Panel格式）
  · 对ALT_ACCOUNTS每个替代解释构建一个Panel
  · 检查：控制后主系数β是否仍显著
  · 输出tables/table4c_alt_accounts.csv

  Table 4d 溢出效应分析（★位置调整：稳健性之后，加分项）
  · 前提：SPILLOVER_VAR已定义（毗邻/关联处理单元）
  · 溢出放在稳健性之后：审稿人先认可识别，再看溢出的拓展意义
  · 判断：同向→扩散效应，反向→替代/位移，不显著→对照组干净
  · 输出tables/table4d_spillovers.txt

步骤6：Table 5 机制分析
  ──────────────────────────────────────
  方式1：渠道变量法（三步回归）
  · 基准：Y ~ X（得到β）
  · 步骤②：M ~ X（得到γ，X对中间变量的效应）
  · 步骤③：Y ~ X + M（得到β'，系数应下降）
  · 传导比例 = (β - β') / β × 100%
  · ★写作规范（保守）：说"结果与该渠道一致"，不说"主要机制""必要机制"
  · ★必须加免责声明："传导比例基于OLS描述性分解，非严格因果估计"
  · β'<β 说明M是渠道；β'>β说明抑制效应，需谨慎

  方式2：异质性调节法（交互项）
  · Y ~ X + M + X×M + controls | FE
  · τ系数方向验证机制预测方向
  · 说"结果与机制预测方向一致"，不说"机制预测成立"

  · 输出tables/table5_mechanism.txt

步骤7：Table 6 异质性分析（Panel格式）
  ──────────────────────────────────────
  · 对每个HETEROGENEITY_VARS构建一个Panel
  · 变量类型判断：
    - 0/1虚拟变量 → 直接分两组
    - 连续变量 → 取中位数分高/低组
  · 表格结构：
    - 行 = 各Panel（异质性维度）
    - 列 = OUTCOME_VARS前5个Y结果变量
    - 每格 = 该子组对应Y的回归系数（含星号和SE）
    - 末列 = 全样本交互项规格的组间差异p值
  · 只报告理论上有方向预测的Panel
  · 输出tables/table6_het_[变量名].csv

步骤8：写 ECO_RESULTS.md
  · 样本描述（N/个体数/时期数）
  · 主模型β/SE/t/效应%/Back-of-Envelope换算
  · 内生性处理结论（平行趋势/IV F统计量）
  · 稳健性通过情况
  · 机制传导比例（每个机制渠道）
  · 异质性结论（哪组效应更强，差异是否显著）
  · 所有表格文件清单

================================================================
七、eco-write-paper：写论文
================================================================

输入：研究主题
读取：ECO_BRIEF + ECO_DATA_PROFILE + ECO_LIT + ECO_RESULTS

写作规范（全部内嵌，不读外部文件）：
  · AER引言8段结构
  · 结论5段结构
  · 计量模型方程（DiD四种变体）
  · 标准表格体系（Table 1-6格式）
  · 数字规范（4位有效数字 + t统计量 + 效应%）
  · 稳健性检验写作模板

论文结构：
  摘要（300字）：背景/问题/方法/主系数/机制/政策含义

  一、引言（1500字，8段结构）
    段1：社会现象/政策背景（开篇引用数字）
    段2：研究问题（X→Y因果）
    段3：识别策略（为何可信，外生性来源）
    段4：数据（规模/来源/时间范围）
    段5：主要结论（β数字 + 效应%）
    段6：机制（传导渠道概述）
    段7：文献贡献（3个角度）
    段8：全文结构

  二、文献综述与假设（1000字）
    · 3-5条脉络（来自ECO_LIT）
    · 现有不足 → 本文贡献
    · H1/H2/H3假设

  三、背景与机制（800字）
    · 制度背景、政策事件时间线
    · 理论机制图（X→M1→Y, X→M2→Y）

  四、数据与实证策略（1200字）
    · 数据来源和处理（含清洗说明）
    · 描述统计（引用Table 1）
    · 计量模型方程
    · 识别假设论证（平行趋势/排他性）

  五、基准结果（1000字）
    · Table 2逐列解读
    · Table 2b多结果Y解读
    · 效应量化（β + 占均值%）

  六、内生性处理（800字）
    · 事件研究图解读（CS-DiD或TWFE，视是否Staggered而定）
    · IV结果（若有）

  七、稳健性检验（800字）

九、设计原则（顶刊导向）
================================================================

1. 切口要小
   · 不是"X影响Y"，是"某个具体事件在特定时间/地点对微观单元的影响"
   · 有明确年份、政策名称、处理组和对照组

2. 识别要干净
   · DiD优先（具体政策冲击）
   · 平行趋势必须验证（事件研究图）
   · 替代解释必须明确排除（Table 4c）

3. 多结果展示广度
   · 同一事件对理论预期相关的多个Y同时检验（Table 2b）
   · 不是盲目扫描，是理论驱动的多维度验证

4. 异质性要理论驱动
   · 不是机械分组遍历
   · 每个Panel对应一个有方向预测的理论机制

5. 机制要具体
   · 3个命名渠道（参考论文：资源租金/机会成本/grievance）
   · 每个渠道有代理变量和数据检验
