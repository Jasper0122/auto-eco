# Eco-Background：政策制度背景研究

## 你的任务

针对研究主题中的具体政策事件，通过 WebSearch 和 WebFetch 搜集第一手资料，输出可直接用于论文 Section 3（制度背景）的结构化文本。

**不依赖模型记忆**——所有政策细节必须来自本次搜索到的具体来源，不可依靠训练数据中的泛化描述。

## 参数解析

- $ARGUMENTS 第一个参数：政策名称或研究主题（如 "宽带中国战略 2015年试点"）
- $ARGUMENTS 第二个参数：项目目录路径

---

## 步骤 1：提取政策关键词

若项目目录存在 `ECO_BRIEF.md`，读取其中的研究主题和识别出的政策事件。

从研究主题中识别：
- **政策正式名称**（用于官方文件搜索）
- **主管部委**（发改委/工信部/财政部等）
- **实施年份范围**
- **覆盖地理层级**（省/市/县/区）

---

## 步骤 2：多轮 WebSearch 搜集原始资料

执行以下6轮搜索，每轮记录找到的关键信息和来源URL：

**第1轮：官方政策文件**
```
[政策正式名称] 实施方案 通知 [主管部委] site:gov.cn
```
目标：找到政策原文或官方解读，确认：起始年份、覆盖范围、申请条件、目标指标

**第2轮：政策批次和名单**
```
[政策名称] 试点名单 批次 [年份] 入选城市/县/企业
```
目标：确认处理组构成（哪些单元入选）、时间分布（是否分批次→是否 Staggered）

**第3轮：政策背景与动机**
```
[政策名称] 出台背景 政策目标 [年份] 改革
```
目标：理解政策设计逻辑，为论文中"为什么这是外生冲击"提供叙事依据

**第4轮：实施细节**
```
[政策名称] 实施进展 效果 [年份区间] 报道
```
目标：了解政策落地情况，识别可能的干扰因素（同期政策、实施延迟等）

**第5轮：学术文献背景引用**
```
[政策名称] institutional background China [DiD/causal] 2019 2020 2021 2022 2023
```
目标：找到已发表论文如何描述该政策背景（可作为描述措辞参考，但引用需核实）

**第6轮：数据与统计**
```
[政策覆盖领域] 统计公报 [年份] [省/市级] 数据
```
目标：找到可引用的官方统计数据，用于背景节中的数量描述

---

## 步骤 3：构建政策时间线

整理搜索结果，建立政策实施时间线：

```
[YYYY年MM月] → [事件/文件名] → [来源]
[YYYY年MM月] → [事件/文件名] → [来源]
...
```

识别关键节点：
- 政策宣布时间（公告/文件发布）
- 政策实施时间（正式生效/试点开始）
- 批次差异（若多批次，各批次时间和规模）
- 政策调整/扩展时间（若有）

---

## 步骤 4：起草 Section 3 正文

按以下结构撰写约 800 字的制度背景节，**每个事实后标注来源**：

```markdown
## 三、制度背景

### 3.1 政策出台背景

[2-3句话：政策出台前的状况，说明"为什么需要这个政策"。
引用官方统计或报告中的数字，如"截至[年份]，全国[指标]仅为[数值]"]
（来源：[URL或文件名]）

[2-3句话：政策设计目标和顶层逻辑]
（来源：[URL或文件名]）

### 3.2 政策内容与实施机制

[3-4句话：政策的核心内容——谁实施，覆盖哪些单元，通过什么机制传导到微观主体]

具体而言，[政策名称]于[年份]由[部委]发布，要求/支持[具体措施]。
[政策覆盖单元数量和地理分布的描述]。
（来源：[URL]）

### 3.3 实施批次与处理组构成

[若有多批次，说明各批次时间、数量、地区分布]
[强调准随机性的叙事：为什么对照组单元未能入选，
以及入选标准的外生性/客观性——这是识别策略叙事的关键]

[批次时间线表格（若适用）]：

| 批次 | 公布时间 | 入选数量 | 地区分布 |
|------|---------|---------|---------|
| 第一批 | [YYYY年] | [N] | [描述] |
| 第二批 | [YYYY年] | [N] | [描述] |

（来源：[URL]）

### 3.4 与本研究的关联

[2-3句话：将政策特征与研究设计对接——
为什么这个政策构成 X（核心解释变量）的外生变化，
为什么对照组单元是合理的反事实]

[1句话：说明数据来源与政策时间窗口的对应关系]
```

## 步骤 5：质量检查

对草稿执行以下检查，不通过则补充搜索：

- [ ] 每个具体数字/日期都有 URL 来源支撑？
- [ ] 说明了处理组如何确定（哪些单元入选，哪些没有）？
- [ ] 有批次时间线（若 Staggered DiD，此项必须）？
- [ ] 提供了识别准随机性的叙事（为什么入选不是内生的）？
- [ ] 提及了同期可能的干扰政策（潜在混淆来源）？

## 注意事项

- **来源优先级**：gov.cn > 人民日报/新华社 > 学术论文背景章节 > 其他媒体
- 若官方文件无法直接访问（404/需登录），记录文件名和发布机构，在输出中标注"文件已失效，建议人工获取"
- 数字若来自模型训练记忆而非本次搜索，**明确标注"[未核实，来自模型记忆]"**，不静默引用
- 背景节字数建议：经济类3区4区期刊约600-1000字，不要超过1200字
